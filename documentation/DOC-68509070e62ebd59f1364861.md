# Planner Tático de Frangpunk - Documentação Técnica

## Resumo da Tarefa

Implementar um planner tático visual e intuitivo para mestres de jogo de Frangpunk, permitindo o planejamento de sessões, esboço de estratégias, posicionamento de Lancers e utilitários, e compartilhamento de planos com jogadores. O planner deve permitir o upload de imagens de mapas, posicionamento de ícones de Lancers e utilitários, e desenho de anotações sobre o mapa.

## Plano de Implementação Técnica

Este plano detalha os passos necessários para implementar o Planner Tático de Frangpunk, conforme a demanda de negócio.

**1. Estrutura de Arquivos e Modificações:**

*   **`index.html`:** Modificar para adicionar os elementos da interface do usuário (UI) para o planner tático. Isso inclui:
    *   Um seletor de mapa (dropdown ou similar) para escolher o mapa base.
    *   Uma paleta de Lancers para selecionar e adicionar ao mapa.
    *   Uma paleta de Utilitários para selecionar e adicionar ao mapa.
    *   Controles para ferramentas de desenho (linha, círculo, etc.).
    *   O canvas onde o mapa, Lancers, utilitários e desenhos serão renderizados.
*   **`style.css`:** Modificar para estilizar os novos elementos da UI, garantindo uma interface intuitiva e responsiva.
*   **`script.js`:** Modificar para adicionar a lógica principal do planner tático. Isso inclui:
    *   Carregar e exibir os mapas a partir do diretório `assets/maps`.
    *   Carregar e disponibilizar os ícones de Lancers e utilitários a partir dos diretórios `assets/lancers` e `assets/utils`.
    *   Implementar a funcionalidade de posicionamento de Lancers e utilitários no mapa, permitindo arrastar e soltar.
    *   Implementar a funcionalidade de desenho, permitindo desenhar linhas, círculos e outros marcadores simples no mapa.
    *   Gerenciar o estado do planner (mapa atual, posições dos Lancers/utilitários, desenhos).
    *   Implementar event listeners para os controles da UI (seletores, botões, canvas).

**2. Lógica Principal (script.js):**

*   **Inicialização:**
    *   Ao carregar a página, carregar a lista de mapas, Lancers e utilitários dos diretórios correspondentes.
    *   Populate os seletores/paletas na UI com os mapas, Lancers e utilitários carregados.
    *   Inicializar o canvas e o contexto 2D.
*   **Gerenciamento de Mapas:**
    *   Adicionar um event listener para o seletor de mapa.
    *   Quando um mapa é selecionado, carregar a imagem correspondente e desenhá-la no canvas.
*   **Gerenciamento de Lancers e Utilitários:**
    *   Adicionar event listeners para os ícones de Lancers e utilitários na paleta.
    *   Ao clicar em um ícone, preparar para posicioná-lo no mapa.
    *   Ao clicar no canvas, desenhar o ícone na posição do clique.
    *   Implementar a funcionalidade de arrastar e soltar para reposicionar os ícones.
*   **Ferramentas de Desenho:**
    *   Adicionar controles para selecionar a ferramenta de desenho (linha, círculo, etc.) e a cor.
    *   Adicionar event listeners para os eventos de mouse no canvas (mousedown, mousemove, mouseup).
    *   Ao usar uma ferramenta de desenho, desenhar no canvas com base nos eventos de mouse.
*   **Responsividade:**
    *   Adaptar o tamanho do canvas para diferentes tamanhos de tela.
    *   Garantir que os elementos da UI sejam responsivos.

**3. Componentes da Interface (index.html):**

*   **Seletor de Mapa:** Um `<select>` ou similar para escolher o mapa.
*   **Paleta de Lancers:** Uma `<div>` contendo imagens dos Lancers, que podem ser clicadas para selecionar e posicionar.
*   **Paleta de Utilitários:** Uma `<div>` contendo imagens dos utilitários, que podem ser clicadas para selecionar e posicionar.
*   **Controles de Desenho:** Botões ou controles para selecionar a ferramenta de desenho e a cor.
*   **Canvas:** Um elemento `<canvas>` para renderizar o mapa, Lancers, utilitários e desenhos.

**4. Estilização (style.css):**

*   Estilizar os novos elementos da UI para garantir uma aparência intuitiva e agradável.
*   Garantir que o planner seja responsivo em diferentes dispositivos.
*   Manter a consistência com o design existente do projeto.
