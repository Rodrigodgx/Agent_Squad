# Tela de Planejamento Tático Interativo para FragPunk

## Resumo da Tarefa

Esta tela permitirá que os jogadores planejem visualmente suas estratégias antes e durante as partidas de FragPunk. O usuário poderá selecionar Lancers e Utilidades disponíveis e arrastá-los para uma representação do mapa, simulando suas posições e movimentos. A tela também oferecerá ferramentas de desenho básicas, como caneta e borracha, para que os jogadores possam adicionar anotações e marcar pontos de interesse no mapa.

**Critérios de Aceite:**

*   O usuário deve conseguir visualizar uma representação do mapa do jogo (estática ou dinâmica).
*   O usuário deve conseguir selecionar Lancers disponíveis (com base na sua conta/seleção de jogo) e arrastá-los para o mapa.
*   O usuário deve conseguir selecionar Utilidades disponíveis (com base na sua conta/seleção de jogo) e arrastá-los para o mapa.
*   O usuário deve conseguir movimentar os ícones de Lancer e Utilidade no mapa, reposicionando-os conforme necessário.
*   O usuário deve ter acesso a uma ferramenta de desenho (tipo caneta) para fazer anotações diretamente no mapa.
*   O usuário deve ter acesso a uma ferramenta de borracha para apagar anotações indesejadas.
*   A interface deve ser intuitiva e fácil de usar, mesmo para jogadores novatos.

### Plano de Implementação Técnica

**1. Análise e Preparação:**

*   **Análise da Estrutura Existente:** O projeto já possui `index.html`, `style.css`, `script.js` e uma pasta `assets` com imagens de mapas, lancers e utilidades. Isso fornece uma base sólida para a implementação.
*   **Definição de Componentes:** Identificar os componentes principais da interface:
    *   Seletor de Mapa: dropdown para escolher o mapa.
    *   Lista de Lancers: lista arrastável de lancers disponíveis.
    *   Lista de Utilidades: lista arrastável de utilidades disponíveis.
    *   Canvas de Planejamento: área principal onde o planejamento tático ocorre.
    *   Barra de Ferramentas: botões para caneta, borracha e outras ferramentas futuras.

**2. Modificações no `index.html`:**

*   Adicionar as seguintes seções ao HTML:
    *   `div` para o Seletor de Mapa, contendo um elemento `select` com as opções de mapas (carregados dinamicamente via JavaScript).
    *   `div` para a Lista de Lancers. Esta lista será populada dinamicamente com elementos `img` para cada lancer, permitindo arrastar e soltar.
    *   `div` para a Lista de Utilidades, similar à Lista de Lancers.
    *   `canvas` com o id `planner-canvas` (já existente) para a área de planejamento.
    *   `div` para a Barra de Ferramentas, com botões para selecionar a ferramenta de desenho (caneta, borracha).

**3. Modificações no `style.css`:**

*   Adicionar estilos para:
    *   Layout geral da tela de planejamento (posicionamento dos componentes: Seletor de Mapa, Listas de Lancers/Utilidades, Canvas, Barra de Ferramentas). Usar flexbox ou grid para o layout responsivo.
    *   Estilos para o Seletor de Mapa (dropdown).
    *   Estilos para as Listas de Lancers e Utilidades (visualização dos itens, efeitos de hover, cursor de arrastar).
    *   Estilos para o Canvas (bordas, tamanho).
    *   Estilos para a Barra de Ferramentas (botões, ícones).

**4. Modificações no `script.js`:**

*   **Carregamento Dinâmico de Mapas:**
    *   Ao carregar a página, ler os arquivos de imagem da pasta `assets/maps/` e preencher o `select` do Seletor de Mapa com as opções.
    *   Ao selecionar um mapa, atualizar a imagem de fundo do `canvas` com o mapa selecionado.
*   **Carregamento Dinâmico de Lancers e Utilidades:**
    *   Ler os arquivos de imagem das pastas `assets/lancers/` e `assets/utils/` e preencher as Listas de Lancers e Utilidades com elementos `img` correspondentes.
*   **Implementação da Funcionalidade de Arrastar e Soltar (Drag and Drop):**
    *   Implementar a lógica de "drag and drop" para permitir que os usuários arrastem os ícones de Lancer e Utilidade das listas para o `canvas`.
    *   Ao soltar o ícone no `canvas`, criar uma nova instância do ícone na posição solta.
*   **Implementação das Ferramentas de Desenho:**
    *   Implementar a lógica para desenhar linhas no `canvas` quando a ferramenta "caneta" estiver selecionada.
    *   Implementar a lógica para apagar linhas no `canvas` quando a ferramenta "borracha" estiver selecionada. Isso pode ser feito através da manipulação dos pixels do canvas ou do rastreamento dos desenhos como objetos.
*   **Gerenciamento de Estado:**
    *   Considerar o uso de um objeto JavaScript para armazenar o estado atual do planejamento (posições dos lancers, utilidades, desenhos). Isso facilitará a implementação de funcionalidades futuras como salvar/carregar planos.

**5. Testes:**

*   Testar a funcionalidade de carregamento de mapas, lancers e utilidades.
*   Testar a funcionalidade de arrastar e soltar.
*   Testar as ferramentas de desenho (caneta e borracha).
*   Verificar a responsividade da interface.

**6. Refinamento:**

*   Refatorar o código para melhorar a legibilidade e manutenção.
*   Otimizar o desempenho da aplicação.
*   Adicionar tratamento de erros.
